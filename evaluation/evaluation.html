Introduction

This website is accessible for people with low vision, since the site has screenreader capabilities trough ARIA landmarks. It also supports the exclusive use of a keyboard, using the tab key to navigate the page, making the page useable for people with a motor dysfunction. Furthermore, the page does not make use of any audio cues nor conveys any information through color, shape or locations, making it accessible for colorbind and deaf people.

Accessibility highlights

The accessibility highlights we deem the most important for our page are: being able to navigate the page exclusively by keyboard (2.1.1), and to have information conveyed through presentation to be programmically determined/available in text (1.3.1). This would greatly benefit people who cannot use the mouse to still access the page, and allows people with limited eyesight to make use of a screenreader which can interpret the page text.

Appendix A

1.1.1 All non-text content that is presented to the user has a text alternative that serves the equivalent purpose. Does not meet the requirement, because the pictures in the table do not have a alternative text.
1.3.1 Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text. Semantic markdown is used for almost everything, making programmatically determining the context of the page possible so it can present this information to users in different modalities however, some ARIA landmarks could be added to differentiate different regions of the page.
1.3.2 When the sequence in which content is presented affects its meaning, a correct reading sequence can be programmatically determined. Meets the requirement: since the visual presentation of the sections does not match the programmatically determined order, but the meaning of the page does not depend on the order of the sections.
1.3.3 Instructions provided for understanding and operating content do not rely solely on sensory characteristics of components such as shape, color, size, visual location, orientation, or sound. Meets the requirement: since the page does not use any shapes or locations to convey information.
1.4.1 Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. Meets the requirement: the page does not specifically use color to convey information.
1.4.2 If any audio on a Web page plays automatically for more than 3 seconds, either a mechanism is available to pause or stop the audio, or a mechanism is available to control audio volume independently from the overall system volume level. Meets the requirement: No audio is used on the page.
2.1.1 All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes, except where the underlying function requires input that depends on the path of the user's movement and not just the endpoints. Meets the requirement: Everything on the page is operable via keyboard. Does not meet the requirement: with the exception of sorting the table.
2.1.2 If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface, and, if it requires more than unmodified arrow or tab keys or other standard exit methods, the user is advised of the method for moving focus away. Meets the requirement: Tab can be used to navigate the page, including the form used for the table, resulting in no keyboard traps.
2.4.1 A mechanism is available to bypass blocks of content that are repeated on multiple Web pages. Meets the requirement: Multiple webpages are not used.
2.4.2 Web pages have titles that describe topic or purpose. Meets the requirement: Main pages has a title that describes the phone store website.
2.4.3 If a Web page can be navigated sequentially and the navigation sequences affect meaning or operation, focusable components receive focus in an order that preserves meaning and operability. Meets the requirement: Focus order is linear from left to right and top to bottom.
2.4.4 The purpose of each link can be determined from the link text alone or from the link text together with its programmatically determined link context, except where the purpose of the link would be ambiguous to users in general. Meets the requirement: Purpose of each link is clear from the link text
